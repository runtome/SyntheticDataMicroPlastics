# SyntheticDataMicroPlastics

Synthetic data generation for microplastics image segmentation. Generates new training images using **patch-based composition** to address class imbalance and increase dataset size for U-Net / Attention U-Net models.

## Problem

The original dataset has severe class imbalance:

| Class    | Annotations | Percentage |
|----------|-------------|------------|
| Fiber    | 625         | 69.37%     |
| Fragment | 230         | 25.53%     |
| Film     | 46          | 5.11%      |

**Total:** 596 images, 901 annotations, 3 classes

## How It Works

The pipeline uses a **cut-paste composition** approach:

1. **Extract** — Individual microplastic objects are extracted from source images using their COCO segmentation masks, producing RGBA patches with soft alpha edges
2. **Background** — Clean backgrounds are generated by inpainting objects out of real microscopy images (or using noise/blank alternatives)
3. **Augment** — Each object patch is randomly augmented: rotation (0-360°), scale (0.5-1.5x), flips, brightness jitter, optional blur
4. **Compose** — 1-4 augmented objects are placed onto backgrounds with overlap control, producing pixel-perfect COCO annotations automatically
5. **Balance** — Class weights (Fiber=1.0, Fragment=2.5, Film=8.0) deliberately oversample minority classes

## Installation

```bash
pip install -r requirements.txt
```

### Requirements

- Python 3.9+
- numpy, opencv-python, scikit-image, albumentations, Pillow, tqdm

## Usage

```bash
# Generate 500 synthetic images (default settings)
python generate_synthetic.py --num-images 500

# Generate 1000 images with heavy Film oversampling
python generate_synthetic.py --num-images 1000 --film-weight 12.0

# Use noise backgrounds instead of inpainting
python generate_synthetic.py --num-images 500 --background-mode noise

# Custom seed for reproducibility
python generate_synthetic.py --num-images 500 --seed 123 --output-dir synthetic_v2
```

### CLI Options

| Flag                 | Default     | Description                                      |
|----------------------|-------------|--------------------------------------------------|
| `--num-images`       | 500         | Number of synthetic images to generate            |
| `--output-dir`       | synthetic_output | Output directory                             |
| `--annotation`       | annotation.json | Path to source COCO annotation file           |
| `--image-dir`        | images      | Path to source images directory                   |
| `--seed`             | 42          | Random seed for reproducibility                   |
| `--min-objects`      | 1           | Minimum objects per synthetic image               |
| `--max-objects`      | 4           | Maximum objects per synthetic image               |
| `--background-mode`  | inpaint     | Background mode: `inpaint`, `crop`, `noise`, `blank` |
| `--max-overlap`      | 0.15        | Maximum overlap ratio between placed objects      |
| `--fiber-weight`     | 1.0         | Sampling weight for Fiber class                   |
| `--fragment-weight`  | 2.5         | Sampling weight for Fragment class                |
| `--film-weight`      | 8.0         | Sampling weight for Film class                    |
| `--num-backgrounds`  | 100         | Number of background images to pre-generate       |
| `--jpg-quality`      | 95          | JPEG output quality                               |

## Method 2: Diffusion Model (DDPM)

A class-label-conditioned denoising diffusion model trained on individual object crops. Generates new microplastic objects from learned distributions rather than copy-pasting real ones.

### Additional Requirements

- PyTorch 2.0+, torchvision, diffusers, accelerate (included in `requirements.txt`)
- GPU recommended for training; CPU works for small experiments

### Training

```bash
# Full training (GPU)
python train_diffusion.py --num-epochs 300 --batch-size 16

# Quick test (CPU)
python train_diffusion.py --num-epochs 5 --batch-size 4 --save-every 5

# Resume from checkpoint
python train_diffusion.py --resume diffusion_checkpoints/checkpoint-100
```

| Flag             | Default                  | Description                         |
|------------------|--------------------------|-------------------------------------|
| `--image-size`   | 128                      | Square crop size in pixels          |
| `--num-epochs`   | 300                      | Training epochs                     |
| `--batch-size`   | 16                       | Batch size                          |
| `--lr`           | 1e-4                     | Learning rate                       |
| `--save-every`   | 50                       | Checkpoint interval (epochs)        |
| `--resume`       | —                        | Checkpoint dir to resume from       |
| `--fp16`         | off                      | Mixed precision (GPU only)          |

### Inference

```bash
# Generate individual crops (saved as PNG with alpha)
python generate_diffusion.py --checkpoint diffusion_checkpoints/checkpoint-300 \
    --num-per-class 50

# Generate full composed scenes with COCO annotations
python generate_diffusion.py --checkpoint diffusion_checkpoints/checkpoint-300 \
    --compose --num-images 200 --background-mode inpaint

# Generate only specific classes
python generate_diffusion.py --checkpoint diffusion_checkpoints/checkpoint-300 \
    --classes Film Fragment --num-per-class 100
```

| Flag                | Default              | Description                              |
|---------------------|----------------------|------------------------------------------|
| `--checkpoint`      | (required)           | Path to diffusers checkpoint directory   |
| `--num-per-class`   | 50                   | Crops to generate per class              |
| `--classes`         | all                  | Subset of classes to generate            |
| `--compose`         | off                  | Enable scene composition mode            |
| `--num-images`      | 100                  | Number of composed scenes                |
| `--background-mode` | inpaint              | Background mode for composition          |

## Recommended Usage for Training

Combine synthetic data with original data at a 1:1 or 2:1 ratio (synthetic:real). Do not replace real data entirely.

```
real : synthetic = 1 : 1  OR  1 : 2
```
